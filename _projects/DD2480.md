I took a software engineering foundamentals course at KTH, where I contributed to 4 team projects, a part of which was issue resolution of a real open-source project! 
Most of the projects were implemented in Java, with the last one being in Python.

## Decide
A decision algorithm which analyzes a sequence of tracking data and outputs a boolean value.

[link](https://github.com/rippyboii/DD2480-DECIDE-V)

## Continuous Integration server
A minimal Continuous Integration (CI) in Java using GitHub webhooks. The server listens to webhook events, checks out pushed commit/pull request, compiles and builds the snapshot, runs the automated test set, and then reports the outcome back to GitHub via commit status using GitHub REST API.

All of the builds are saved in a database, allowing for a search of past test runs.

[link](https://github.com/a-runebou/DD2480-CI-V)

## Refactoring, coverage, cyclomatic complexity
In this assignment we chose the Apache Commons Imaging library, identified functions with low coverage and high cyclomatic complexity, refactored them to make them simpler, and improved test coverage by implementing new tests.

[link](https://github.com/a-runebou/DD2480-CCC-V/tree/report)
